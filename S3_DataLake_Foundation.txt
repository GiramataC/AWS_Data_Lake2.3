DEVIATION FROM LAB SPEC
--------------------------------------------------------------------
1. Bucket was NOT created from scratch. It already existed
   (data-lake-prod-463470950988), provisioned by Lab 2.1's Terraform as an
   unavoidable prerequisite. This lab's Terraform (AWS_Lab2.3) references it
   via a data source and only adds the pieces that were still missing:
   bucket policy, access logging, CloudTrail, curated/temp/archive folders,
   lifecycle rules. The four extra tags (Environment/Owner/Purpose/
   CostCenter) required a small edit to Lab 2.1's s3_bucket.tf, since tags
   live on the bucket resource itself.

2. Bucket policy does not implement a hard "deny everyone except these 3
   roles" rule. A strict denylist-of-other-principals would also have
   blocked this account's own Terraform/admin role and DataSyncS3Role
   (Lab 2.1), risking a real management lockout. Instead: two Deny
   statements (enforce HTTPS, enforce SSE on upload) apply to everyone, and
   two Allow statements grant DataEngineerRole/GlueServiceRole/
   RedshiftIAMRole explicit bucket + object access.

3. Region is eu-west-1, not us-east-1 as in the generic lab doc, to stay
   consistent with every prior lab (1.1, 1.2, 2.1, 2.2) in this account.
